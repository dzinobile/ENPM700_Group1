On-vehicle pedestrian detection, ground-plane projection, tracking, and safety alerts
v1 • Oct 21, 2025
Overview

Mission & Scope

  • Ingest live monocular video; undistort via calibrated camera model.
  • Project pedestrian footpoints to ground plane using planar homography.
  • Track IDs frame-to-frame in ground coordinates; cull far outliers.
  • Emit braking alerts if any track within D_close.

Approach

Architecture (Pragmatic Split)

  • Classes: PerceptionApp, VideoSource/OpenCVCamera, CameraModel, GroundProjector, HumanDetector, Tracker, SafetyManager.
  • POD: BBox, Detection, Measurement, Track (value types, no virtuals).
  • Free funcs: geometry & utility helpers (e.g., footpoint, distance).
Risks

Key Risks & Mitigations

  • Calibration drift / wrong extrinsics → self-check with reprojection error bounds; on-fleet sanity tests.
  • Detector false positives → score threshold + temporal smoothing; future: pose or keypoints.
  • Planar assumption break → curb ramps/slope: guard with ROI mask + horizon gating.
  • Frame drops / latency → lock-free capture ring; drop-oldest policy to keep real-time.

Outcomes

Metrics & Deliverables

  • Latency: < 33 ms/frame @ 30 FPS (720p target).
  • Range gating: remove tracks beyond D_max; alert if min(range) < D_close.
  • ID stability: > 90% ID retention across 3 seconds, moderate motion.
  • Artifacts: UML (class + sequence), config schema, unit tests, demo video.
